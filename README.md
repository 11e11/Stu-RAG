## 🚀 项目介绍

高校教务信息分散在毕业要求、选课手册、成绩政策等各类文档中，学生提问口语化、语义模糊，直接检索召回率低。为此基于 Spring Boot 实现教务智能问答平台，覆盖**多租户知识库管理、异步文档入库、并发控制与多轮智能问答**全流程。

- **多路检索**：稠密 + 稀疏混合检索，多通道并行召回，RRF 融合去重重排，兼顾精准匹配与语义理解。
- **意图识别**：树形多级分类，置信度不足主动引导澄清，教务场景下自动路由到对应知识域。
- **异步入库**：基于 RocketMQ 异步解耦文档入库任务，Pipeline 节点编排（解析→分块→向量化→写入），教务政策文档按条款粒度独立分块。
- **并发控制**：基于 Redisson 分布式信号量控制 LLM 调用并发上限，并发满时请求进入 Redis 公平等待队列排队执行，超时未获取许可则快速拒绝。
- **多轮会话**：滑动窗口 + 自动摘要压缩历史上下文，学生可连续追问教务问题，系统保持上下文连贯。

## 🏗️ 核心架构

采用前后端分离的架构模式，后端按职责分为四个 Maven 模块。

`framework` 层提供通用基础能力，`bootstrap` 层专注教务业务逻辑。



## ✨ 功能特性

| 特性 | 说明 |
|:---|:---|
| **限流** | 基于 Redisson 分布式信号量控制 LLM 并发上限，公平排队 + 超时拒绝 |
| **熔断** | 模型健康检查 + 失败计数，自动熔断不可用的模型，配合降级链自动切换 |
| **可观测性** | 基于 AOP 的全链路 Trace，每个环节的耗时、输入输出、异常信息都有记录 |
| **流式输出** | SSE 实时推送，首包探测机制保证模型切换时用户无感知 |
| **会话管理** | 记忆压缩、摘要持久化、TTL 过期，多轮对话不丢上下文 |
| **异步入库** | RocketMQ 异步解耦文档入库，Pipeline 节点编排，入库过程可监控 |
